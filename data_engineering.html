Data Engineering Projects

Projects focused on building scalable data pipelines, ETL processes, data modeling, and automation using Python, SQL, Spark, and other tools.

Anime Catalogue Data Pipeline (Apache Airflow, Python, PostgreSQL)

Developed an automated ETL (Extract, Transform, Load) pipeline to collect, clean, and store anime metadata from a public API for downstream analysis and reporting. The pipeline extracts data from the Jikan API, which provides information about top anime series, including titles, airing dates, episodes, duration, broadcast schedule, and ratings. The workflow ensures the data is accurate, standardized, and readily accessible for analytics.
The pipeline produces a clean, structured anime catalogue CSV that is automatically loaded into Postgres daily. This setup enables efficient querying, analysis, and reporting on anime trends, airing schedules, and ratings, demonstrating expertise in data engineering, ETL automation, and workflow orchestration.

NYC Green Taxi Trip Data Pipeline (Apache Airflow, Python, PostgreSQL)

Designed and implemented an automated ETL pipeline to extract, transform, and load NYC Green Taxi trip data into a PostgreSQL database for analysis of trip patterns, distances, and fares. The pipeline downloads monthly taxi trip data, cleans and filters it, and loads it into a structured database, enabling data-driven insights into passenger counts, trip distances, and fare amounts.
The pipeline produces a clean, query-ready NYC Green Taxi dataset that is automatically updated daily in PostgreSQL. This enables efficient analysis of trip metrics and supports further data science and reporting efforts, demonstrating expertise in production ETL pipelines, data cleaning, and workflow automation.

Healthcare ETL Pipeline (Apache Airflow, Python, PostgreSQL)

Designed and implemented an automated ETL pipeline to extract, transform, and load CMS Outpatient Claims data into a PostgreSQL database for analysis of claim amounts, provider activity, and patient payment ratios. The pipeline downloads compressed CSV data from the CMS public repository, extracts and cleans the dataset, calculates key metrics such as patient responsibility ratio and claim duration, and loads the processed data into a structured database for analytics.
The pipeline produces a clean, query-ready outpatient claims dataset that can be efficiently queried in PostgreSQL, enabling reporting, dashboard creation, and data-driven insights into healthcare service utilization and billing patterns. This demonstrates expertise in building production-grade ETL pipelines, data cleaning, workflow automation, and large-scale data handling.
Technologies used: Apache Airflow, Python (pandas, io, zipfile, base64), PostgreSQL, SQLAlchemy, Docker, HTTP APIs
